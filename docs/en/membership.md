# Membership

Membership should support both learning and responsibility.

## Member Types

- **Members** participate in public learning, open-source practice, and regular projects.
- **Core members** help review work, maintain repositories, manage recruitment, and protect infrastructure.
- **Maintainers** are responsible for specific repositories, projects, or workflows.

## Expectations

Members are expected to:

- collaborate respectfully;
- avoid leaking sensitive materials;
- document meaningful work;
- disclose AI-assisted contributions when relevant;
- ask for review before changing important project direction.

## Access

Access should follow least privilege. Ordinary members may access public repositories and normal project repositories. Sensitive governance, recruitment, infrastructure, and unreleased project repositories require limited access.
