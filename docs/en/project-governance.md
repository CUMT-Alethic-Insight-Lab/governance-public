# Project Governance

Projects should remain small enough to understand and serious enough to maintain.

## Starting a Project

Before creating a new project repository, define:

- purpose;
- expected users;
- maintainers;
- public or private visibility;
- sensitive materials that must not be published;
- expected deliverables.

## Repository Rules

Unless a repository states otherwise, organization-level defaults from `.github` apply.

If a repository deviates from organization defaults, its README should explain the difference.

## Maintainers

Each active project should have at least one maintainer who can review Pull Requests, triage Issues, and keep documentation current.
